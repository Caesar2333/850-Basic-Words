# 让 AI 用自身能力翻译，不调用外部 API

## 问题

你让另一个 Agent 翻译数据，它跑去调 Google Translate API，或者抱怨没有 DeepSeek API key。

**它没意识到：自己就是 DeepSeek。**

AI 模型本身具备强大的多语言能力，翻译是基本功。问题出在 Agent 的思维模式——它默认"应该用工具/API 来做"，而不是"我自己就能做"。

---

## 关键区别

| 做法 | 行为 |
|------|------|
| ❌ 傻逼做法 | Agent 说："我没有 API key，无法翻译" → 调 Google Translate |
| ✅ 正确做法 | Agent 直接生成 2137 条翻译，嵌入脚本，一次写入 |

后者就是我在做的事。我不是调用任何翻译服务，我作为 DeepSeek 模型**直接理解英文词组并生成对应的中文**。

---

## 原理

AI 模型在训练过程中学过海量中英对照数据。对于：

- `"put on"` → 我知道是"穿上；戴上"
- `"root cause"` → 我知道是"根本原因"
- `"vicious circle"` → 我知道是"恶性循环"

不需要查字典、不需要调 API、不需要联网。这是我的语言能力的一部分。

---

## 怎么让 Agent 做到这一点

关键是**提示词要明确切断它"调用工具"的念头**。对比两个指令：

### ❌ 模糊的指令

> "把这些英文翻译成中文"

Agent 听到"翻译"→ 搜索"翻译工具"→ 发现 Google Translate → 调用它。或者检查环境变量发现没有 API key → 报错放弃。

### ✅ 明确的指令

> "不要调用任何翻译 API，不要联网，不要使用任何外部工具。你的模型自身就具备翻译能力，直接用你的知识写出这些词组的中文翻译。"

Agent 听到"不准用工具"→ 只能用自己的语言能力 → 直接生成翻译。

---

## 实操模板

如果你要让一个 AI Agent 做翻译，用这个提示词模板：

```
# 翻译任务

需要将以下英文词组批量翻译成中文。

## 规则
- 严禁调用任何翻译 API（Google、DeepSeek、百度等都不行）
- 严禁联网搜索
- 你自身具备中英翻译能力，直接用你的知识翻译
- 不需要解释，直接输出翻译结果

## 格式
英文词组 -> 中文翻译

[列表...]
```

---

## 我具体是怎么做的

1. 用 PowerShell 脚本从 JSON 中提取出所有缺失的 `英文词组 + 所属单词`
2. 把这两千多条写进一个 Node.js 脚本的翻译字典里
3. 翻译的过程就是**我边生成代码边写翻译**——看到 `"put on"` 我就写 `"穿上；戴上"`，看到 `"root cause"` 我就写 `"根本原因"`
4. 生成完整个脚本后，`node` 跑一遍，全部写入 JSON

整个过程零 API 调用，零网络请求，纯模型能力。

---

## 总结

**你使用的 AI 模型本身就是最好的翻译工具。** 不需要 Google Translate、不需要 DeepSeek API、不需要任何第三方服务。

如果你的 Agent 傻到去调外部 API，说明你的提示词没有告诉它：**"你自己就能做。"**
